Frontend Development

  • Frontend development, or "front-end," is a discipline in web and software development that focuses on creating visually appealing and user-friendly user interfaces (UI) and experiences (UX). Frontend developers concentrate on the client side, dealing with what users interact with in their browsers. A good UI effectively presents information, while UX encompasses the overall positive and efficient user interaction with a product. Web pages are composed of HTML for content structure, CSS for styling, and JavaScript for interactivity. CSS layouts define the structure and arrangement of elements on a web page, determining their position, size, and relationships. In programming, languages like JavaScript serve as a secret code to instruct computers, enabling users to teach their computers to perform cool and fun tasks by providing step-by-step instructions that the computer follows.

Backend Development

  • An API (Application Programming Interface) is a set of protocols and tools that enables developers to interact with the backend services of a platform. It provides a way for external applications to access and utilize the features of the system programmatically. APIs allow third-party developers to integrate platform functionality into their applications, fostering a broader ecosystem of interconnected services and enhancing user experiences beyond the official platform. Common HTTP methods, such as GET for data retrieval, POST for submitting content, and DELETE for removing resources, are typically utilized by APIs. These methods enable developers to interact programmatically with the backend, facilitating actions like accessing data, posting content, and managing interactions.

Server Hosting and Deployment

  • A server is a system that delivers services to clients in a network, and hosting involves providing the infrastructure for websites or applications to be accessible online. Web hosting companies offer server space for storing website files, databases, and assets. Deployment is the process of making a software application or system available for use.

Server-side vs. client-side

  • Client-side code executes on the user's computer, downloaded and run by the browser when a web page is viewed. It's responsible for the page's display. In contrast, server-side code runs on the server, and its outcomes are downloaded and displayed in the browser, handling processes on the server side. JavaScript is employed in various contexts, functioning both on the client side for interactive web elements and on the server side for backend operations.

Dependencies

  • In software development, dependencies refer to external modules or libraries that a project depends on for specific functionalities, promoting code reuse and augmenting capabilities. Generally sourced from third parties, these components streamline development by eliminating the necessity to construct everything from the ground up.

Debugging

  • Debugging is the method of recognizing and rectifying errors or bugs in a computer program. It encompasses tracing and isolating issues within the code, comprehending their origins, and implementing corrections to guarantee the program operates as intended. Debugging is a vital aspect of software development to improve code quality and ensure the accurate execution of a program. Developers employ diverse tools and techniques, such as breakpoints, logging, and step-by-step execution, to pinpoint and resolve bugs in their code.

Version Control System

  • A Version Control System (VCS) is a software tool created to ease the management of alterations to source code or any set of files over time. It is extensively used in collaborative software development to track and synchronize changes made by different contributors working on a project.

GIT (Github)

  • GIT is a decentralized version control system enabling collaboration among multiple developers on a project. GitHub, on the other hand, is a web-based platform offering Git repository hosting. It extends Git's capabilities through a graphical interface, collaboration tools, and supplementary features.
